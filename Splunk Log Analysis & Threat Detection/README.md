### 🎯 Objective
Practise enterprise-grade log analysis using Splunk SPL to detect security threats, build monitoring dashboards, and simulate full incident investigations from alert to remediation.

### 🛠️ Tools Used
- **SIEM:** Splunk (Free/Trial)
- **Query Language:** SPL (Search Processing Language)
- **Log Sources:** Windows Event Logs, Syslog, Authentication Logs
- **Framework:** MITRE ATT&CK

### 🔧 What I Did
- Ingested 5,000+ security logs from Windows and Linux sources
- Wrote SPL queries to detect:
  - Brute-force login attempts
  - Lateral movement patterns
  - Suspicious process execution
  - Failed authentication spikes
- Built 5+ Splunk dashboards:
  - Top 10 alert types (pie chart)
  - Failed logins over time (line chart)
  - Alerts by severity (bar chart)
  - Source IP geolocation map
  - User activity timeline
- Investigated 3 simulated incidents from initial alert through to remediation
- Documented findings in structured incident reports

### 📊 Key Findings
- Detected simulated brute-force attack within 2 minutes of onset
- Identified lateral movement pattern across 3 endpoints
- Reduced mean time to detect (MTTD) through optimized SPL queries
